1. Project Overview

The Repricer v2 front-end is a modern React application built to manage and monitor product repricing operations. It provides an intuitive interface for viewing product data, managing algorithm settings, and monitoring errors.

Core Features

  • Product Management: Browse and manage products with advanced filtering and sorting
  • Algorithm Configuration: Configure repricing algorithm settings per vendor
  • Real-time Updates: Monitor product status and price changes
  • Error Tracking: View and manage unhandled errors
  • Responsive Design: Mobile-first approach with full desktop support
Note: This application is part of a monorepo structure and integrates with a backend API for data operations.

2. Frontend Architecture

Architectural Patterns

Component-Based Architecture

The application follows a component-based architecture with clear separation of concerns:

  • Page Components: Top-level route components that orchestrate layout and data fetching
  • Feature Components: Reusable business logic components (forms, tables, navigation)
  • UI Components: Generic, reusable UI primitives from shadcn/ui

File-Based Routing

Utilizes TanStack Router's file-based routing system for automatic route generation:

  • Routes are defined in src/routes/ directory
  • Auto-generated route tree provides type-safe navigation
  • Nested layouts supported via __root.tsx

State Management Strategy

Hybrid approach combining:

  • React Hooks: Local component state with useState/useEffect
  • TanStack Table: Table state management (sorting, filtering, pagination)
  • React Hook Form: Form state and validation
  • Client-side Caching: Manual cache implementation with timestamps

Data Fetching Pattern

const fetchProducts = async () => {
  try {
    setIsLoading(true);
    setError(null);

    const response = await fetch("/productV2");

    if (!response.ok) {
      throw new Error(`HTTP error! status: ${response.status}`);
    }

    const result = await response.json();

    if (result.success && result.data) {
      setData(result.data);
      setCacheTimestamp(Date.now());
    } else {
      throw new Error(result.message || "Failed to fetch products");
    }
  } catch (err) {
    setError(err.message);
    toast.error("Failed to load products");
  } finally {
    setIsLoading(false);
  }
};

Data Update Pattern

const onSubmit = async (data: SettingsFormData) => {
  try {
    const response = await fetch(
      `/v2-algo/${mpId}/${vendorId}`,
      {
        method: "PUT",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify(data),
      }
    );

    if (!response.ok) throw new Error("Update failed");

    toast.success("Settings updated successfully");
  } catch (err) {
    toast.error(err.message);
  }
};

Error Handling

  • HTTP Errors: Checked via response.ok
  • Network Errors: Caught in try-catch blocks
